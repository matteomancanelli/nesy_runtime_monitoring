TOSEM (Borges et al.) vs IJCNN2015 (Perotti et al.) — Key Differences
Both papers share Artur as a common author and both combine neural networks with temporal logic for a verify-then-adapt loop, but they operate at fundamentally different levels.
The TOSEM paper works at the model level. It starts with a system description (a NuSMV transition system — essentially a set of next-state rules describing how the world evolves), verifies whether this model satisfies temporal properties (in CTL/LTL over infinite traces), and when verification fails, uses the counterexample to adapt the model via a NARX recurrent network. The learning engine modifies the transition system itself so that it eventually satisfies the desired properties. The key idea is: "I have a broken system description, I want to repair it guided by properties." The temporal logic properties are fixed; the model changes.
The IJCNN2015 paper (and the HCVS2014 paper) works at the property/specification level. RuleRunner encodes a temporal formula (in FLTL, i.e., LTL over finite traces) as a set of Horn-clause rules, then translates those rules into a neural network via the CILP algorithm. The neural network is the monitor — it performs runtime verification by checking whether an observed trace satisfies the property. The adaptation step (IJCNN2015) then modifies the property encoded in the neural network: given traces that the domain expert says are misclassified, the system learns an adapted property that reclassifies them correctly and extracts the new formula. The key idea is: "I have a specification that doesn't match reality, I want to adapt the specification guided by traces."
So in short: TOSEM adapts the system model to satisfy fixed properties; IJCNN2015 adapts the property/specification to match observed system behavior.

The core contribution: DFA-based monitoring vs. rule-based monitoring
RuleRunner builds a monitor syntactically from the formula's parse tree. For each operator it creates evaluation rules and reactivation rules, then encodes them as a neural network via CILP. The number of rules is linear in the formula structure, and each rule fires based on local truth values. The three-way comparison:

(1) Purely symbolic DFA monitoring. You compile φ to a DFA and track the state explicitly, using standard automata operations. No neural network at all.
